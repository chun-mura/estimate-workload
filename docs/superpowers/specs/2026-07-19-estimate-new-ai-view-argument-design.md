# `estimate:new` AI支援表示オプション設計

## 目的

`/estimate:new` 実行時に、AI支援を考慮した工数見積もりを引数で明示的に
指定できるようにする。指定された場合は、AI支援表示の対話確認を行わない。

## 引数契約

要件ソースの前に、次の先頭オプションを指定できる。

- `--mode quality` または `--mode economy`
- `--ai-view`（AI支援表示を含める）
- `--no-ai-view`（AI支援表示を含めない）

これらは順不同で併用できる。例:

```text
/estimate:new --mode quality --ai-view OAuthログインを追加する
/estimate:new --no-ai-view docs/requirements.md
```

`--ai-view` と `--no-ai-view` の併用、同一オプションの重複、未知の先頭
オプション、無効な `--mode` 指定は、解析・エージェント起動・CALC 実行の前に
入力エラーとして停止する。

## 処理フロー

1. `--mode` と AI表示オプションを検証して除去する。
2. 明示的な AI表示指定があれば、その値を採用する。
3. AI表示指定がなければ、従来どおり対話で確認する。
   - モード未指定時は、モードとAI表示を一回の質問で確認する。
   - モード指定済み時は、AI表示のみを確認する。
4. AI表示を含める場合のみ、AI支援係数の参照を読み込む。

既存の `pipeline` ペイロード、レポート、および AI支援係数の扱いは変更しない。
